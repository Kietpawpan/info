(ร่าง) ข้อเสนอโครงการพัฒนาระบบ e-Tracking Backend

MNRE-SLC-012

ระบบ e-Tracking Backend

1. หลักการและเหตุผล

ในปัจจุบัน (ปีงบประมาณ พ.ศ. 2567) การปรับปรุง (update) ข้อมูลสถานะของคำขอข้อมูลข่าวสารในระบบติดตามคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking) ยังใช้วิธีให้ผู้พัฒนาระบบ (Frontend Developer) ดำเนินการแก้ไข source code ของฐานข้อมูลสถานะคำขอ ซึ่งต้องใช้ความรอบคอบและระมัดระวังอย่างยิ่ง หากให้บุคคลอื่นดำเนินการแทน อาจไปลบหรือเติมรหัสคอมพิวเตอร์คลาดเคลื่อน ซึ่งจะทำให้ระบบ e-Tracking เสียหายทันที

ดังนั้น เพื่อป้องกันเหตุดังกล่าว และเพื่อให้มีระบบสำหรับเจ้าหน้าที่ผู้ปฏิบัติงานแต่ละหน่วยงาน เพื่อที่จะสามารถทำการปรับปรุงข้อมูลลงระบบติดตามสถานะคำขอได้ โดยไม่ต้องอาศัยความรู้ความชำนาญด้าน Coding สำนักงานปลัดกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม จึงจัดทำโครงการพัฒนาระบบสนับสนุนการปรับปรุงฐานข้อมูลของระบบติดตามสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking Backend) เพื่อให้มีระบบ Backend สำหรับใช้ควบคุมการแก้ไข source code ของฐานข้อมูลของระบบ e-Tracking โดยไม่ต้องเข้าถึงเครื่องแม่ขายจากระยะไกลแต่อย่างใด จะทำให้การ update ฐานข้อมูลดังกล่าวเป็นไปอย่างสะดวก รวดเร็ว และเรียบร้อย


2. วัตถุประสงค์ของโครงการ

เพื่อพัฒนาระบบสนับสนุนการปรับปรุงฐานข้อมูลของระบบติดตามสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking Backend)


3. วิธีดำเนินงาน

การพัฒนาระบบหลังบ้านของระบบติดตามสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking Backend) เพื่อสนุบสนุนเจ้าหน้าที่ผู้ปฏิบัติงาน ให้สามารถ Update สถานะคำขอในระบบ e-Tracking โดยไม่ต้องใช้ความรู้และทักษะด้าน Coding เริ่มจากผู้พัฒนาระบบ Backend ดำเนินการพิจารณาศึกษาทำความเข้าใจรหัสคอมพิวเตอร์ต้นฉบับ (source code) ของระบบ e-Tracking ว่าประกอบด้วยไฟล์ใด เก็บที่โฟลเดอร์ใดบนเครื่องคอมพิวเตอร์แม่ข่าย และมีลักษณะการจัดเรียงข้อมูลในภาษาใด รูปแบบใด แล้วพิจารณาออกแบบ/เขียนโปรแกรมหลังบ้าน เชื่อมโยงระบบฐานข้อมูลหลังบ้านกับระบบ Update ซึ่งเป็นหน้าต่างโปรแกรม สำหรับเจ้าหน้าที่เปิดเข้าใช้งานเพื่อปรับปรุงแก้ไขฐานข้อมูลสถานะคำขอในไฟล์นั้นเป็นการภายใน ทั้งนี้ ระบบ Backend จะรับข้อมูลหมายเลขประจำตัวประชาชน (13 หลัก) ของผู้ขอข้อมูลข่าวสาร และทำการเข้ารหัสข้อมูลดังกล่าวตามมาตรฐานสากลในฝั่งผู้ใช้งาน (client side) เพื่อคุ้มครองข้อมูลส่วนบุคคลของผู้ขอข้อมูลข่าวสาร รวมถึงรับข้อมูลสถานะคำขอเป็นข้อความ (text) ที่ผ่านกระบวนการตรวจสอบความถูกต้องของข้อมูลนำเข้า (input validation) ก่อนเริ่มกระบวนการแปลงข้อมูลนำเข้าทั้งหมด ให้เป็นรหัสฝั่งแม่ข่าย (server-side script) แล้วออกคำสั่งเรียกดูรหัสข้อมูลทั้งหมดในฐานข้อมูลของระบบ e-Tracking แล้วดำเนินการตัดแต่งรหัสข้อมูลในไฟล์ฐานข้อมูลดังกล่าว ตามข้อมูลนำเข้าที่ได้เข้ารหัสไว้ แล้วต่อเติมรหัสข้อมูลใหม่กลับเป็นฐานข้อมูลดังเดิม (คล้ายการผ่าตัดเปลี่ยนถ่ายอวัยวะภายในร่างกายคน) เมื่อระบบได้ดำเนินการ update เสร็จสิ้นแล้ว ให้มีการแจ้งเตือนเจ้าหน้าที่ผู้ใช้งานทราบต่อไป ในการนี้ จะมีการประยุกต์ใช้ความรู้และทักษะ ดังนี้

  • การเขียนโปรแกรมฝั่งผู้ใช้งานและฝั่งแม่ข่าย (Full Stack Devloper) โดยใช้ทักษะ Coding ในภาษา HTML/CSS/JavaScript/PHP
  • มาตรฐานแอปลิเคชันภาครัฐสำหรับอุปกรณ์เคลื่อนที่ (Government Mobile Application Standard)
  • ทฤษฎีการออกแบบสื่อ (Gestal Theory) และการออกแบบบริการอิเล็กทรอนิกส์ (Service Design)
  • การเขียนผังงาน (Flowchart) ของโปรแกรมคอมพิวเตอร์ ตามมาตรฐาน IBM
  • พระราชบัญญัติข้อมูลข่าวสารของราชการ พ.ศ. 2540 และพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562
  • การเข้าถึงเครื่องแม่ข่ายของรัฐและรักษาความลับของทางราชการมิให้รั่วไหล
  • การทดสอบโปรแกรมด้วยอุปกรณ์คอมพิวเตอร์บนระบบปฏิบัติการที่หลากหลาย ได้แก่ Windows/Androi/iOs/Mac
  • การขอคำปรึกษาจากผู้เชี่ยวชาญด้านการเขียนโปรแกรมคอมพิวเตอร์ในต่างประเทศ ผ่านแพลตฟอร์มต่าง ๆ
